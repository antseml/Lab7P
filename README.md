# Лабораторная работа №6

## Задание
Доработать программу из лабораторной работы №6

**Необходимо выполнить следующие требования:**
- Организовать хранение коллекции в реляционной СУБД (PostgresQL). Убрать хранение коллекции в файле.
- Для генерации поля id использовать средства базы данных (sequence).
- Обновлять состояние коллекции в памяти только при успешном добавлении объекта в БД
- Все команды получения данных должны работать с коллекцией в памяти, а не в БД
- Организовать возможность регистрации и авторизации пользователей. У пользователя есть возможность указать пароль.
- Пароли при хранении хэшировать алгоритмом SHA-512
- Запретить выполнение команд не авторизованным пользователям.
- При хранении объектов сохранять информацию о пользователе, который создал этот объект.
- Пользователи должны иметь возможность просмотра всех объектов коллекции, но модифицировать могут только принадлежащие им.
- Для идентификации пользователя отправлять логин и пароль с каждым запросом.

**Необходимо реализовать многопоточную обработку запросов:**
- Для многопоточного чтения запросов использовать создание нового потока (java.lang.Thread)
- Для многопотчной обработки полученного запроса использовать создание нового потока (java.lang.Thread)
- Для многопоточной отправки ответа использовать ForkJoinPool
- Для синхронизации доступа к коллекции использовать java.util.Collections.synchronizedXXX

## Доступные команды в клиентском приложении
| Команды | Описание |
|---------|----------|
| add                   | Создать новый маршрут и добавить его в коллекцию|
| execute_script {файл} | Считать и исполнить скрипт из указанного файла |
| show                  | Вывести все маршруты |
| clear                 | Удалить все маршруты |
| update {ID}           | Обновить маршрут по ID |
| remove_by_id {ID}     | Удалить маршрут из коллекции по его ID |
| add_if_max            | Создать новый маршрут и добавить в коллекцию, если его значение превышает значение наибольшего элемента этой коллекции |
| history               | Вывести последние 7 команд (без их аргументов) |
| remove_lower          | Удалить из коллекции все маршруты, меньшие, чем определенный маршрут |
| help                  | Вывести справку по доступным командам |
| exit                  | Завершить программу |
| print_descending      | Вывести маршруты в порядке убывания |
| filter_contains_name {подстрока} | Вывести маршруты, названия которых содержат указанную подстроку |
| average_of_distance   | Вывести среднее расстояние всех маршрутов |
| info                  | Вывести информацию о коллекции маршрутов (тип, дата инициализации, количество элементов) |

> На сервере можно использовать команду **save** для сохранения коллекции (**доступно только на сервере!**)

## Структура проекта
- **client** - клиентская часть приложения
- **server** - серверная часть приложения
- **common** - *"базовый"* модуль: содержит описание запросов/ответов, структуру маршрутов и т.д. (все что необходимо для работы и сервера, и клиента)
- **build** - клиентское и серверное приложения, файл с данными для запуска сервера

## Инструкция по сборке и запуску

Приложение собирается в исполняемые `.jar` архивы. Перед запуском убедитесь, что база данных PostgreSQL доступна.
